    API de configuración de anuncios SSAI de Video Cloud

    En este tema, aprenderá a utilizar la API de inserción de anuncios en el lado del servidor (SSAI) para crear y administrar configuraciones de anuncios.

    Las configuraciones de anuncios definen varios aspectos de la reproducción SSAI, incluidas las llamadas publicitarias, los indicadores y otras opciones de configuración. Una cuenta puede tener varias configuraciones y actualmente las configuraciones no se pueden compartir entre cuentas.

    Puede crear, enumerar y eliminar fácilmente configuraciones de anuncios desde Video Cloud Studio. Para obtener más información, consulte el documento Configuración de la configuración de anuncios del lado del servidor .
    Notas:
    • Para utilizar SSAI, su cuenta de Video Cloud debe estar configurada para Entrega dinámica y habilitada para SSAI. Póngase en contacto con su administrador de cuentas para empezar a usar esta función.
    • SSAI funciona con Vídeo bajo demanda (VOD) y admite la reproducción en vivo.
    • Para obtener una lista de limitaciones, consulte el documento Visión general de SSAI .

    Información general

    La siguiente información se refiere a todas las solicitudes de API SSAI.

    URL base

    La URL base para la API SSAI es:

    https://ssai.api.brightcove.com/v1

    Ruta de la cuenta

    En todos los casos, se solicitarán una cuenta de Video Cloud específica. Por lo tanto, siempre agregará el término seguido de su ID de cuenta a la URL base:accounts

    https://ssai.api.brightcove.com/v1/accounts/your account id

    Autenticación

    La autenticación para solicitudes requiere un encabezado de autorización:

    Authorization: Bearer your access token

    El es un token de acceso temporal de OAuth2 que debe obtenerse del servicio OAuth de Brightcove.access_token Para obtener información detallada sobre cómo obtener credenciales de cliente y usarlas para recuperar tokens de acceso, consulte la descripción general de OAuth de Brightcove.

    Operaciones

    Cuando solicite credenciales de cliente, deberá especificar el tipo de acceso a la cuenta u operaciones que desea. A continuación se muestra una lista de las operaciones actualmente soportadas para la API SSAI:

    • Datos de SSAI:

      video-cloud/ssai/read
      video-cloud/ssai/all

    Administrar configuraciones de anuncios

    La API admite las siguientes solicitudes GET y PUT:

    • Configuraciones de anuncios de lista
    • Crear una configuración de anuncio
    • Obtener detalles de configuración de anuncios
    • Actualizar una configuración de anuncio

    Configuraciones de anuncios de lista

    Enumera las configuraciones de anuncios definidas para una cuenta.

    Método CONSEGUIR
    URL https://ssai.api.brightcove.com/v1/accounts/su ID de cuenta/ssai_configs
    Encabezados Autorización: Portador su token de acceso
    Tipo de contenido: aplicación/json

    Encabezados Autorización: Portador su token de acceso
    Tipo de contenido: aplicación/json

    Cuerpo de la muestra:

    {
  "name": "VMAP Ad Server",
  "vmap_response_namespace": "bc",
  "ad_config": {
  	"expected_ad_response": "vast_3_0",
  	"disable_server_beacons": false,
    "round_up_cue_points": false,
  	"template_url": {
  		"template": "http://solutions.brightcove.com/bcls/ads/simple-ad-rules?ip={{ system.ip_address }}&vid={{ metadata.video_id }}&ppid={{ system.unique_user_id }}&p_vmap={{ url.p_vmap }}"
  	}
  },
  "discontinuities": {
    "hls": [ "*" ]
  }
}

    Detalles del campo de configuración

    Esta tabla define los campos de configuración de anuncios utilizados en la sección de cuerpo de una solicitud.

    Campo Tipo Descripción
    name Cadena Nombre legible por humanos
    vmap_response_namespace Cadena Ajusta la salida de VMAP para que utilice el formato de espacio de nombres Unicorn Once heredado o para utilizar el nuevo formato de espacio de nombres de Brightcove.

    Valores: uo, bc Predeterminado: bc
    description Cadena Descripción legible por humanos
    ad_config.expected_ad_response Cadena Qué tecnología usar para analizar la respuesta. [ 1]

    Valores:
    • dfp_ad_rules
    • dfp_vmap
    • smart_xml
    • vast_3_0
    ad_config.disable_server_beacons Booleano Marca si se deshabilita el disparo del lado del servidor de impresiones o balizas de anuncios

    Cuando se establece en true, SSAI no activará ningún indicador del lado del servidor e incluirá todos los indicadores en la salida de VMAP

    Cuando se establece en false, SSAI activará las balizas que puede incluir en el servidor e incluir cualquier que no pueda en la salida de VMAP
    ad_config.round_up_cue_points Booleano Marca si se redondea al siguiente fotograma clave al elegir una posición cercana para insertar anuncios.

    Valor predeterminado: false, que elige el fotograma clave más cercano a la posición del anuncio deseada.
    ad_config.template_url.template Cadena Plantilla de etiqueta de anuncio. Variables disponibles descritas en una sección posterior.
    ad_config.template_url.defaults Objeto Asignación de cadena a cadena que define el valor predeterminado para usar en el caso en que no se proporciona un parámetro url.

    Puede ser un literal { "url.foo": "SomeString" }

    O hacer referencia a otra variable { "url.foo": "{{ metadata.title_id }}" }
    discontinuities.dash [ 2] [] Cadena Controla qué versiones de guión se entregan manifiestos de guión de varios períodos.

    Establezca ["*"] para entregar guión de varios períodos para todas las versiones Lista

    vacía para nunca

    Ejemplo: ["live-timeline"] para entregar para la línea de tiempo en vivo pero no para hbbtv
    discontinuities.hls [ 2] [] Cadena Controla qué versiones de hls entregar con discontinuidades.

    Establezca ["*"] para entrega con discontinuidades en todas las versiones de la lista HLS

    Empty for never

    Ejemplo: ["v4","v5"] para entregar para v4 y v5 pero no para v3
    beacon_templates Array Una matriz de balizas para disparar (ejemplo: balizas de terceros)
    beacon_templates[].type Cadena

    Tipo de baliza para disparar. Valores:

    • content_start
    • content_first_quartile
    • content_midpoint
    • content_third_quartile
    • content_complete
    • content_quartiles
    • content_interval
    • ad_start
    • ad_first_quartile
    • ad_midpoint
    • ad_third_quartile
    • ad_complete
    • ad_quartiles
    • ad_break_start
    • ad_break_end
    • segment_start
    • segment_end
    • on_load
    beacon_templates[].template_url.template Cadena Plantilla URL de baliza
    extend_beacon_guard_ttl Booleano Establece la duración del TTL de Beacon Guard (Tiempo de vida) en la duración del TTL de sesión del contenido. De lo contrario, el valor predeterminado es 1 minuto.

    Las siguientes notas hacen referencia a los campos de la tabla anterior.

    [ 1] Cuando expected_ad_response se establece en vast_3_0 , SSAI realiza una llamada VAST por cada punto de referencia de anuncio definido en Video Cloud. Para VMAP y reglas publicitarias, SSAI realiza solicitudes basadas en los saltos de anuncio definidos en la respuesta inicial al anuncio.

    [ 2] Las etiquetas de discontinuidad indican un cambio en la codificación de la velocidad de fotogramas entre los segmentos de vídeo de contenido y anuncio. Le dicen al sistema que reconstruya la cadena de renderizado. Para SSAI escalable, el reproductor de vídeo necesita poder reproducir HLS con etiquetas de discontinuidad y MPEG DASH que contenga varios periodos.

    Variables de anuncio

    Las variables en la url de la plantilla se identifican mediante llaves dobles ({{ … }} ) con espacios en blanco opcionales antes y después de la ruta de la variable. Todas las variables están precedidas por uno de los tres espacios de nombres:

    • sistema
    • url
    • metadatos

    Variables de sistema

    Las variables de sistema son proporcionadas por el sistema SSAI y pueden ser información sobre el usuario final o variables auxiliares para generar valores aleatorios. Los valores deben estar codificados con URI antes de insertarse en las plantillas de URL.

    Las variables del sistema se identifican como: {{system.*}}

    Campo Descripción
    ip_address Dirección IP del usuario final
    random_number_32 Entero aleatorio de 32 bits
    random_guid UUID aleatorio
    referer Valor del encabezado Referer del usuario final
    timestamp_utc Hora actual como marca de tiempo Unix
    unique_user_id MD5 (dirección_ip_+ user_agent)
    unix_timestamp Hora actual como marca de tiempo Unix (segundos)
    user_agent Valor de encabezado User-Agent del usuario final
    uuid Uuid aleatorio
    x_forwarded_for Valor de encabezado X-Forwarded-For del usuario final
    xfp.correlator Entero aleatorio de 64 bits
    xfp.ip_address Dirección IP del usuario final
    xfp.unique_user_id MD5 (dirección_ip_+ user_agent)
    xfp.scor Entero aleatorio de 64 bits

    Variables de URL

    Los parámetros de consulta proporcionados en el punto de entrada VMap/manifiesto están disponibles en el espacio de url nombres. A diferencia de las variables bajo otros espacios de nombres, estos parámetros no están codificados por url cuando se insertan en la plantilla. Si los valores de las variables deben estar codificados en url yendo al proveedor de anuncios, deberán estar codificados con doble url en la url del punto de entrada.

    Las variables URL se identifican como: {{url.*}}

    Variables de metadatos

    Las variables de metadatos son aquellas que describen el vídeo de contenido, derivado de orígenes de datos de Video Cloud y Dynamic Delivery. Los valores están codificados por URL antes de ser insertados en las plantillas de URL.

    Las variables de metadatos se identifican como: {{metadata.*}}

    Campo Descripción
    ad_keys Cadena de texto de forma libre que se puede agregar y editar en el módulo multimedia de Video Cloud Studio
    custom_fields.{field_name} Campos personalizados de Video Cloud
    long_description Descripción larga de Video Cloud
    name Nombre de vídeo de Video Cloud
    reference_id ID de referencia de Video Cloud
    tags Lista separada por comas de las etiquetas de Video Cloud del vídeo
    title.duration Duración del contenido en segundos
    title.id ID de título de entrega dinámica
    title.name Nombre del título de entrega dinámica
    video_id ID de vídeo de Video Cloud
    Otros pares clave/valor de Video Cloud también estarían aquí

    Parámetros de URL de punto de entrada

    Hay un puñado de parámetros de consulta que se pueden agregar a la URL del punto de entrada SSAI (VMAP o manifiesto) para ajustar algunos comportamientos:

    Parámetros Descripción
    ?rule=sd-only Filtra cualquier representación de vídeo que tenga una altura inferior al umbral establecido en la configuración de la cuenta
    ?rule=discos-enabled Habilita la reproducción con discontinuidades en HLS y MultiPeriodos en Dash. Tiene prioridad sobre la configuración de discontinuidades en la configuración de reproducción
    ?rule=discos-disabled Deshabilite la reproducción con discontinuidades en HLS y MultiPeriodos en Dash. Tiene prioridad sobre la configuración de discontinuidades en la configuración de reproducción

    Notas de configuración

    1. La precarga de anuncios no se debe hacer con SSAI. La razón de esto es que si precarga el reproductor informará de una impresión de anuncio y probablemente las primeras balizas de cuartil antes de que se reproduzca el vídeo. Esto podría dar lugar a análisis publicitarios inexactos. Si configura SSAI en Studio, esto se hará automáticamente, pero si configura SSAI manualmente, debe ser consciente de este problema.
    2. Si el reproductor web utiliza SSAI, y una de sus motivaciones para hacerlo es evitar los bloqueadores de anuncios, debe usar balizas del lado del servidor. Las balizas del lado del cliente no deben usarse, ya que se bloquearán.

    Macros del lado del cliente

    Utilice el page prefijo cuando desee utilizar macros de anuncios del lado del cliente. Estas macros permiten utilizar variables en las direcciones URL de servidor y VMAP. Para obtener más información sobre las macros de anuncios, consulte la sección Macros de anuncios y ServerURL del documento Publicidad con el complemento IMA3.

    Las macros del lado cliente tienen el prefijo: {{page.*}}

    Por ejemplo, para agregar la variable document.referrer y una ventana pageVariable DOM, debería prefijarla con el prefijo page en la plantilla de anuncio de la siguiente manera:

    https://adserver.com/{{page.document.referrer}}/{{page.pageVariable.whateverIwant}}

    La API de reproducción devuelve document.referrer y pageVariable.whateverIwant se agrega a las URL de VMAP y SRC. A continuación, el reproductor de Brightcove ejecuta su lógica de reemplazo de macros del lado del cliente para reemplazar los valores apropiados, antes de enviar la solicitud:

    https://bolt-prefix/blah.vmap?document.referrer={document.referrer}&pageVariable.whateverIwant={pageVariable.whateverIwant}

    Balizas de error de anuncio

    Los indicadores de error de anuncios VAST al usar SSAI pueden ser útiles para encontrar y solucionar problemas proactivamente con el flujo de trabajo de anuncios. Para obtener más información, consulte el documento Balizas de error de anuncio con SSAI .
