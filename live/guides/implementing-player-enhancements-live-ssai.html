    Implementación de Mejoras de Jugador para SSAI en Vivo

    En este tema, aprenderá a crear una transmisión en vivo con anuncios del lado del servidor que utilice Mejoras de jugador para SSAI en vivo para mejorar la experiencia del usuario.

    Introducción

    Las mejoras de reproductor de Brightcove para SSAI en directo le permiten mejorar las interrupciones de anuncios en las transmisiones en directo con recuentos de anuncios, temporizadores de cuenta regresiva de anuncios, anuncios de clic a través de anuncios y anuncios complementarios.

    La información de su anuncio se especifica mediante una configuración de anuncio que recupera las respuestas VAST para identificar qué anuncios se mostrarán. El archivo VAST también contiene información sobre anuncios complementarios y URL de balizas.

    Si no está familiarizado con SLS, consulte la descripción general: Mejoras del jugador para el documento SSAI en vivo .

    Flujo de trabajo

    Para crear una transmisión en vivo con SSAI y mejoras de reproductor, siga estos pasos:

    1. Pida a su administrador de cuenta que habilite su cuenta para retransmisiones en directo, inserción de anuncios en el lado del servidor (SSAI) y Mejoras de jugador para SSAI en vivo.
    2. Crear una configuración de anuncio
    3. Opcional: Crear recursos de pizarra para llenar el tiempo de anuncio no utilizado
    4. Crear un trabajo en vivo
    5. Crear un vídeo remoto
    6. Configurar el reproductor
    7. Comience a transmitir
    8. Activar anuncios utilizando puntos de referencia (esto se puede hacer antes o durante el evento en vivo)
    9. Limpieza

    Crear una configuración de anuncio

    Una configuración de anuncios define la información necesaria para incluir anuncios del lado del servidor con una transmisión en directo. Esto incluye la URL del proveedor de anuncios, el tipo de respuesta de anuncio esperado y si se deben enviar los indicadores.

    Para SLS, las configuraciones de anuncios DEBEN tener un ad_configuration_expected_response_type valor de Vast.

    Para crear una nueva configuración de anuncio, envíe una solicitud de la siguiente manera:POST

    Método POST
    URL https://api.bcovlive.io/v1/ssai/applications
    Encabezado X-API-KEY: your API KEY
    Tipo de contenido: aplicación/json

    Todas las solicitudes se autentican mediante una clave API que se le proporciona cuando se configura su cuenta. La clave se pasa en el X-API-KEY encabezado.

    Para obtener más información sobre los campos del cuerpo de la solicitud, consulte el documento Referencia de API de Live .

    Con Mejoras del Reproductor para SSAI en vivo, los campos resaltados se pueden establecer dentro del application_ad_configuration objeto.

    Cuerpo de solicitud de muestra:

    {
  "application_ad_configuration": {
	"ad_configuration_description": "your ad config description",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "your ad server",
    "ad_configuration_client_sdk_enabled": true,
    "client_options": {
      "show_ad_break_remaining_time": true / false,
      "show_ad_remaining_time": true / false,
      "show_number_of_remaining_ads": true / false,
      "client_only_tracking": true / false
    }
  },
  "application_description": "your aaplication description"
}

    Todos los client_options son opcionales. Sólo client_only_tracking tiene un valor predeterminado de false.

    Crear un trabajo en vivo

    Para crear un trabajo activo, envíe una POST solicitud de la siguiente manera:

    Método POST
    URL https://api.bcovlive.io/v1/jobs
    Encabezados X-API-KEY: your API KEY
    Tipo de contenido: aplicación/json

    Para obtener más información sobre los campos del cuerpo de la solicitud, consulte el documento Referencia de API de Live .

    Para un trabajo activo con SSAI habilitado, establezca ad_insertion: true en el cuerpo de la solicitud HTTP.

    Si desea crear un clip VOD a partir de la transmisión en vivo, incluya el videocloud parámetro en el cuerpo de la solicitud. Para obtener más información, consulte la API Live: Creación de documento de clips VOD .

    Desde la respuesta de la API, tome nota del trabajo id y las stream_url propiedades. Los necesitarás más tarde.

    En el cuerpo de la respuesta, habrá un objeto que contendrá información de SSAI para cada configuración de anuncio de su cuenta. En el ejemplo anterior se muestra el objeto de una configuración de anuncio. Debe ser el ID de configuración del anuncio que creó en el paso anterior.

    los playback_url o playback_url_dvr El campo asociado con su ID de configuración de anuncios se usa en el siguiente paso para crear un recurso de video remoto.

    Crear un vídeo remoto

    En Video Cloud, creará un vídeo remoto que apunta a la transmisión en directo. Para este paso, necesitará el playback_url o playback_url_dvr del paso anterior.

    Hay dos formas de crear un video remoto:

    • Usar Video Cloud Studio
    • Usar la API de CMS

    Configurar el reproductor

    El proceso de reproducción de la transmisión en directo desde la API de reproducción es ligeramente diferente de la forma en que reproduce un vídeo normal de Video Cloud. Además de proporcionar el ID de vídeo (id de referencia), debe incluir un token de reproducción que identifique la configuración del anuncio que desea utilizar. También tendrá que especificar si desea reproducir DVR.

    Crear el token de reproducción

    Para crear el token de reproducción, envíe una POST solicitud de la siguiente manera:

    Método POST
    URL https://api.bcovlive.io/v1/jobs/your live job id/playback-token
    Encabezados X-API-KEY: your API KEY
    Tipo de contenido: aplicación/json

    Cuerpo de solicitud de muestra:

    {
   "dvr":true / false,
   "application_id":"your ad config id"
}

    Respuesta de muestra:

    {
 "playback_token": "live.JhQabcdefghijkl000000000abcdefghi"
}

    El cuerpo de respuesta incluirá un token de reproducción generado. Utilícelo cuando realice una llamada a la API de reproducción con los siguientes reproductores:

    Reproductor web

    Para obtener más información sobre la configuración del Reproductor de Brightcove, consulte el documento Using Player Enhancements for Live SSAI with Brightcove Player .

    Reproductor nativo de Android

    Para obtener más información sobre cómo configurar el Reproductor nativo para Android, consulte el documento Using Player Enhancements for Live SSAI con los SDK nativos .

    Reproductor iOS nativo

    Para obtener más información sobre cómo configurar el Reproductor nativo para iOS, consulte el documento Using Player Enhancements for Live SSAI con los SDK nativos .

    Limpieza

    Cuando la transmisión en vivo esté completa, puede limpiar los artefactos de su transmisión de la siguiente manera:

    1. Detener el codificador de transmisión

    2. Cancela el trabajo en vivo.

      El trabajo activo sólo está disponible durante el tiempo de reconexión especificado cuando se creó el trabajo.

       

      Para cancelar un trabajo activo, envíe una PUT solicitud de la siguiente manera:

      Método PUT
      URL https://api.bcovlive.io/v1/jobs/your live job id/cancel
      Encabezados X-API-KEY: your API KEY
      Tipo de contenido: aplicación/json

      Para obtener más información, consulte el documento de referencia de Live API .

    3. Eliminar la configuración del anuncio

      Para eliminar una configuración de anuncio, envíe una solicitud de la siguiente manera:DELETE

      Método PUT
      URL https://api.bcovlive.io/v1/ssai/applications/application/your ad config id
      Encabezados X-API-KEY: your API KEY
      Tipo de contenido: aplicación/json

      Para obtener más información, consulte el documento de referencia de Live API .
